# Node Spec — Pets

> Canonical. **V1 node.** Global rules from `00_README_and_Changelog.md` apply.
> Note: all household-specific assumptions removed (D15) — no fixed pet count or names.

## 1. Purpose

Pets manages household pet care, reminders, records and routines: flea/worming/vaccination
treatments, vet visits, medication, food routines, documents and care instructions. It serves
households that want only basic reminders as well as those wanting detailed history.

## 2. Philosophy

A dedicated node because pet care has recurring reminders, profiles, documents, health-like
records, calendar events and kiosk relevance. Answers: *"What do we need to know or do for the
pets?"* Useful with just a pet name and one reminder.

## 3. What belongs

Pet profiles; flea/worming/vaccination dues; vet appointments; medication schedules; food
notes; microchip numbers; insurance details; pet-sitter instructions; vet contacts; weight
logs; pet documents.

## 4. What does NOT belong

Pet-food **stock** → Inventory. General grocery items → Atlas (unless generated by Inventory).
Vet invoices → attachments linked to Pets. Rewarded "feed the pets" tasks → Meridian (may be
generated from Pets). Human medical records → Health.

## 5. Primary users

Admins/managers manage all pet info. Users view and complete permitted pet-care actions.
Children interact with simple kiosk views (feeding, treatment reminders, pet-care tasks).
Future guest/house-sitter accounts may see limited care instructions.

## 6. Key features

**Pet profiles** — name, species, breed (optional), photo/avatar, birth/adoption date
(optional), colour, notes, vet details, microchip number, insurance details, food notes.

**Treatments** — type (flea, worming, vaccination, medication, grooming, other), name,
`last_done_at`, `next_due_at` (source of truth), `recurrence_rule`, reminder timing, notes,
history. Calendar event via the scheduling helper (`calendar_event_id`).

**Vet appointments** — date/time, location, provider, notes, attachment links, calendar
integration.

**Medication** — name, dosage, frequency, start/end date, notes, reminder schedule.

## 7. Permissions

Generally household-visible; specific records may be restricted (household ·
manager/admin only · user_restricted · sensitive). Children see only simple pet-care tasks and
safe info.

## 8. Hub integration

Widgets: pet reminders due · upcoming vet appointments · medication due · pet care today · pet
profile cards · low pet-food alert (via Inventory).

Completion rule: this node is not "done" until its Hub widget rows are seeded with
`source_node` set, kiosk support declared per widget, and content is supplied through
permission-filtered selectors that the Hub service calls without cross-node model imports.

## 9. Calendar integration

Events for flea/worming/vaccination/vet/medication/grooming, optional feeding routines — all
via the scheduling helper from the owning record's date.

## 10. Notifications

Treatment due soon · treatment overdue · vet appointment upcoming · medication due · pet food
low (via Inventory).

## 11. Events (signals)

Publishes: `pet_created`, `pet_treatment_due`, `pet_treatment_completed`,
`pet_appointment_created`, `pet_medication_due`, `pet_care_task_created`.
Consumes: `inventory_item_low`, `meridian_task_completed`, `travel_trip_created`.
Example: Travel creates a trip → Pets prompts the user to create pet-sitter instructions.

## 12. Search

FTS over pet names, vet details, treatment names, notes, appointment notes, attachment
metadata — permission-filtered.

## 13. Attachments

Vet records, vaccination certificates, insurance documents, photos — via the shared service.

## 14. Kiosk

Highly visual: pet cards, photos, today's pet care, large "done" buttons, upcoming reminders,
simple feeding/treatment instructions. Children should not need to type.

## 15. Data model

`pets`, `pet_treatments` (`next_due_at`, `recurrence_rule`, `calendar_event_id`),
`pet_appointments` (`calendar_event_id`), `pet_weight_logs`. All inherit `HouseholdBaseModel`.
Shared: attachments, calendar_events, notifications, audit_logs.

## 16. V1 scope

Pet profiles · treatment reminders · vet appointments · calendar integration · Hub widget ·
kiosk care view · attachments · basic permissions · FTS. (Works for any number of pets.)

## 17. Future enhancements

Feeding schedules · food-inventory link · weight graphs · pet insurance tracking ·
house-sitter pet mode · medication history · photo gallery · Meridian task generation ·
travel pet-care planning.

## 18. Completion criteria

Users create pets, add treatment reminders, mark treatments complete, see pet reminders on Hub
and Calendar, use kiosk pet cards, ship Hub widget rows/selectors, and search permitted pet
records.
